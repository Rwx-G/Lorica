## Update (2026-04-29, post-review)

VERIFY-001 was closed during the review by adding a mechanical CI gate :
`lorica/src/proxy_wiring/tests.rs::dispatch_order_locked` parses
`proxy_wiring.rs` at compile time via `include_str!`, extracts every
`self.check_<name>(` call site in source order (comments + prose
references filtered out), and asserts the sequence equals the
canonical 16-helper order. Any future reorder, drop, rename, or
addition of a `check_<name>` helper fails the test with a named
regression message that points the maintainer at the PRD AC #8 +
CHANGELOG entry. The test runs in `cargo test -p lorica` alongside
the existing 268 tests (now 269) and adds zero runtime cost to
non-test builds. CHANGELOG `[Unreleased]` entry filed under "Changed".

The original review text below is preserved unchanged for
traceability ; the gate file `docs/qa/gates/8.1-foundation-module-split.yml`
reflects the post-fix state (quality_score 100, top_issues empty).

## Executive Summary

Story 8.1 is a deep, behaviour-preserving refactor of the two largest files in the `lorica` crate (`main.rs` 4329 LOC + `proxy_wiring.rs` 5440 LOC pre-extraction = 9769 LOC) into focused per-concern submodules under `lorica/src/{cli, startup, proxy_wiring, proxy_wiring/filters}/`. The work shipped as 23 commits on `feat/v1.6.0`, plus 2 closing doc commits, plus a `main` merge for v1.5.4. All 8 acceptance criteria are marked Done with documented rebalance / method departures. End-to-end verification (file structure + symbol counts + dispatch order + helper presence + regression-suite presence) confirms the PRD claims line up with the codebase reality.

The gate is **PASS**. Two low-severity concerns are documented for transparency, neither is blocking:

- **VERIFY-001** : AC #8 verification method substituted source-level call-site parity for the prescribed `cargo expand` byte-equality (justified : module-path aliasing across the split makes byte-equal expand output unrealistic; the substitute is materially stronger).
- **DOCS-001** : no `docs/stories/story-8.1-*.md` file exists; the story lives canonically in `docs/prd/epic-8-v1.6.0.md` + `docs/backlog.md` + `CHANGELOG.md` `[Unreleased]`. Future Epic 8 stories should pick a convention (PRD-only vs scaffold per-story files).

## Code Quality Findings

### Positives

- **Zero noise in new code** : no `TODO` / `FIXME` / `dbg!` / `unimplemented!` / `todo!` / commented-out blocks in any of the new `cli/`, `startup/`, `proxy_wiring/filters/`, `proxy_wiring/{lb, helpers, context, routing, error_pages}.rs` files. The two `eprintln!` / `println!` matches in `cli/unban.rs` + `cli/rotate_key.rs` are intentional CLI user-facing output. The one `eprintln!` in `startup/logging.rs:64` is the early-boot logger fallback before the tracing subscriber is initialized - intentional.
- **Three `#[allow(dead_code)]` markers** are accounted for : `startup/logging.rs:25` and `main.rs:1755` are pre-existing (carried through the extraction); `proxy_wiring/routing.rs:38` on the `RouteEntry` struct is for fields read via reflection-shaped patterns the compiler doesn't trace (pre-existing convention from `proxy_wiring.rs` before the extraction). None are new debt.
- **`.cargo/audit.toml`** is well-documented : the three `RUSTSEC-2026-0098/0099/0104` advisories are scoped to the optional `route53` feature path (AWS SDK -> aws-smithy-http-client 1.1.12 -> rustls 0.21 -> webpki 0.101.7), with paragraph-length rationale per advisory and a documented revisit trigger ("on every cargo update touching the AWS SDK"). Matches CLAUDE.md's "new RUSTSEC advisories that hit only the route53 feature path go to `.cargo/audit.toml`" policy.
- **Three v1.5.2-style asymmetries closed in passing** : the structural shape of the extraction (per-concern spawn helpers that force role-explicitness) caught and fixed three bugs that the PRE-extraction code shape was actively hiding. This is the highest-value outcome of the refactor and validates the AC #1 rebalance decision.
- **Decision builder API** : `Decision::reject(status, reason).with_html(html).with_header("Retry-After", "1")` is materially nicer to read at the call site than the previous five-step inline `extract_host` -> `render_error_body` -> `ResponseHeader::build` -> `insert_header(Content-Type)` -> `insert_header(Content-Length)` -> `write_response_header` + `write_response_body` -> `Ok(true)` pattern. The fluent builder also makes per-site policy explicit (the pre-extraction state had only 1 site carrying Retry-After and 1 carrying X-RateLimit-Reset, with the rest silently dropping them; CHANGELOG entry confirms the audit H-10 drift is now closed).
- **M-22 cert-hot-reload regression suite** : the three white-box tests pin the v1.5.2 bug shape (regression pin), the active-cert filter (orphan certs stay unloaded), and the gate short-circuit (out-of-order commits do not overwrite fresher state). The required `IncomingCommand::for_test()` constructor in `lorica-command/src/rpc.rs` is marked `#[doc(hidden)]` to keep production code on the normal `RpcEndpoint::new` dispatch path - good hygiene.

### Concerns (low-severity, non-blocking)

- **VERIFY-001 (low)** : AC #8 verification method substituted source-level call-site count parity for the prescribed `cargo expand` byte-equality check. The substitution is justified in the PRD AC #8 notes ("expansions are equivalent up to module path aliasing, which would require a normalising step that's brittle"). The combined evidence (9 symbol counts + dispatch order + 268-test green at HEAD and at `bd4219b`) is materially stronger for behaviour-preservation than a normalised-expand diff would be (a normalised-expand diff would not catch a behaviour change inside a function whose signature stayed the same; the symbol count + dispatch order check would). Document the substitution as the new convention for future epics that touch the same surface.
- **DOCS-001 (low)** : no `docs/stories/story-8.1-foundation-module-split.md` story file exists. The story lives canonically in `docs/prd/epic-8-v1.6.0.md` + `docs/backlog.md` (items #7 and #8 in "Closed in v1.6.0 cycle") + `CHANGELOG.md` `[Unreleased]` (12 entries). Future Epic 8 stories (8.2..8.12) should pick a convention. Recommended : if 8.2..8.12 follow the `docs/stories/` per-story-file convention from Epics 1-7, retroactively scaffold `story-8.1-foundation-module-split.md` so it can be referenced by 8.2 as a baseline.

### Observations (informational)

- **Test count attribute drift** : `grep -c '#\[test\]\|#\[tokio::test\]'` returns 290 attribute occurrences across 13 files in `lorica/src/`, vs 268 runnable-tests claimed by the PRD. The 22-count drift is normal (cfg-gated tests not built in default features, helpers, doc-tests, nested mod blocks). Not a concern; final `cargo test -p lorica` run will confirm the runnable count.
- **`proxy_wiring.rs` reduction (3690 LOC) is better than the PRD's ~4350 LOC target** by ~660 LOC. The PRD's "~4350" estimate was conservative; the actual extraction was more thorough. Positive drift.
- **`main.rs` final LOC (3566) is 10 LOC under the PRD's 3576** target. Cumulative drift across 5 cleanup commits between the AC text writing and the final commit. Not material.
