## Feature 1 - Connection pre-filter (Phase 1.1)

**Scope:** IP allow/deny CIDR enforced at TCP accept, before TLS handshake. Hot-reloadable.

| AC | Given | When | Then | Tests |
|----|-------|------|------|-------|
| 1.1 | An empty allow list | Any IP connects | Connection is accepted | `connection_filter_test::allow_any_when_empty_policy` |
| 1.2 | An allow list containing 10.0.0.0/8 | An IP in 10/8 connects | Connection is accepted | `connection_filter_test::allow_permits_matching_cidr` |
| 1.3 | An allow list containing 10.0.0.0/8 | An IP outside 10/8 connects | Connection is dropped pre-TLS | `connection_filter_test::allow_denies_non_matching_cidr` |
| 1.4 | A deny list containing 192.168.1.1/32 | That IP connects | Connection is dropped even if also in allow list (deny wins) | `connection_filter_test::deny_always_wins` |
| 1.5 | A running listener with filter policy A | Policy is hot-reloaded to B | Subsequent accepts use B without rebuilding the listener | `connection_filter_test::midstream_hot_reload_applies_new_policy` |

**Coverage: 5/5 AC, no gaps.**

## Feature 2 - Cache predictor (Phase 1.2)

**Scope:** 16-shard LRU remembers keys whose origin responded uncacheable; short-circuits cache state machine.

| AC | Given | When | Then | Tests |
|----|-------|------|------|-------|
| 2.1 | A fresh predictor | A key is looked up | `is_bypassed` returns false | `predictor.rs::test_predictor_lookup_miss_returns_false` |
| 2.2 | A predictor | A key is recorded as bypass | Subsequent lookup returns true | `predictor.rs::test_predictor_records_and_reads_bypass` |
| 2.3 | A predictor at capacity | New keys are recorded | Oldest entries are evicted (LRU) | `predictor.rs::test_predictor_lru_eviction` |

**Coverage: 3/3 AC. GAP: transient-error reasons (InternalError, UpstreamError) are NOT remembered; tested implicitly via only-the-allowed-reasons path (covered in proxy_wiring integration, not in predictor unit tests).**
